图片文件上载

图片文件上载介绍

HTTP 文件上载是指通过 HTTP 协议将用户提交的文件上传到服务器。HTTP 文件上载通常使用 POST 方法，将文件作为请求体发送。

在 HTTP 文件上载中，用户需要使用 Web 界面上传文件。通常，Web 界面会提供文件选择框和上传按钮等控件，以便用户选择要上传的文件，并指示上传的 URL 或文件名。用户可以选择文件后，点击上传按钮，将文件上传到服务器。

HTTP 文件上载的基本原理是将文件作为一个 HTTP 请求体发送。HTTP 请求体通常包括文件的内容、文件名、文件类型等信息。服务器接收到请求后，会根据请求中的内容、文件名等信息，找到并读取文件，并将其存储到服务器上相应的位置。

multipart/form-data格式

multipart/form-data 是一种用于在 HTTP 请求中传输二进制数据的标准格式。通常，它用于在 Web 表单中上传文件，但也可以用于传输其他类型的二进制数据。

当你提交包含文件上传的表单时，浏览器通常会使用 multipart/form-data 编码类型。这种编码类型允许将表单数据以多个部分（parts）的形式传送，每个部分都有自己的内容类型（Content-Type）和一些额外的描述信息。

multipart/form-data 格式的请求主体通常由多个部分组成，每个部分包含一个表单字段的数据。每个部分由一个包含头部信息和实际数据的块组成。头部信息包含了 Content-Disposition、Content-Type 等字段，用于描述该部分的属性

MutipartFile接口

在java中，MutipartFile是Spring框架提供的一个接口，用于处理文件上传。

MutipartFile接口定义了用于访问上传文件的方法，它通常用于处理通过HTML表单上传的文件。这个接口的实现类可以包含上传的内容、元数据和一些额外的信息。在处理文件上传时，Spring MVC控制器可以使用MutipartFile对象来获取上传的文件数据。

MutipartFile接口的一些常见方法：

  • getOriginalFilename():返回上传文件的原始文件名
  • getContentType():返回上传文件的内容类型
  • getSize():返回上传文件的大小，单位为字节
  • getBytes():返回上传文件的字节数组
  • getInputStream():返回一个用于读取上传文件内容的InputStream
  • isEmpty():检查上传文件是否为空
  • transferTo():用于将上传的文件内容保存到指定的目标文件(file)中的方法

在Spring Boot中实现图片文件上载功能

  1. 首先，在配置文件中定义相对应系统类型的上传文件存储路径，以及允许上传的图片格式后缀(写在相应的applicaiton-…yml中)，这些配置可以通过@Value注解到业务层

    # 文件上传路径
upload:
  dir: D:/coolshark_pms/upload
  #LINUX系统
  # dir:  /opt/coolshark_pms/upload

  2. 实现图片文件上传业务层相关接口以及实现类

    /**
 * 文件上传接口
 */
public interface IUploadService {
    /**
     * 文件上传
     * @param path 文件存储路径
     * @param file 文件对象
     * @return 文件的访问地址
     */
    UrlVO uploadFile(String path, MultipartFile file);
}

    (说明：urlVO类是向前端返回该图片文件的url路径，用于前端展示图片内容，MultipartFile接口用于处理前端传来的图片文件)

    @Service //Spring注解 表示将此类注入spring容器并由spring管理
@Slf4j
public class UploadServiceImpl implements IUploadService {
    //定义日期格式化对象 用于生成保存文件的目录
    private static final DateTimeFormatter DATE_TIME_FORMATTER =DateTimeFormatter.ofPattern("yyyy/MM");
    // 保存文件的路径
    @Value("${upload.dir}") //属性注入 内容来自配置文件
    private File uploadDir;
    @Override
    public UrlVO uploadFile(String path, MultipartFile file) {
        //定义保存文件的目录
        File dir = new File(uploadDir,path);
        //获取当前的日期
        String dateDir = LocalDate.now().format(DATE_TIME_FORMATTER);
        // 包含日期的文件保存路径
        File dataDirFile = new File(dir,dateDir);
        log.info("文件保存路径:{}",dataDirFile.getAbsolutePath());
        //创建目录
        if(!dataDirFile.exists()){
            dataDirFile.mkdirs(); //如果不存在 就创建目录
        }
        //获取文件的原始名 
        String originalFilename = file.getOriginalFilename();
        //获取文件的扩展名
        String suffix = originalFilename.substring(originalFilename.lastIndexOf("."));
        //重新生成文件名 UUID会生成64位随机
        String fileName = UUID.randomUUID()+suffix;
        //创建文件
        File dest = new File(dataDirFile,fileName);
        log.info("文件保存路径:{}",dest.getAbsolutePath());
        //保存文件
        try {
            //将前端传来的file内容复制到dest文件中
            file.transferTo(dest);
            //返回文件的访问地址
            String url = "/"+dir.getName()+"/"+dateDir+"/"+fileName;
            return new UrlVO(url);
        } catch (IOException e) {
            log.error("保存文件失败",e);
            throw new FileUploadException("保存文件失败");
        }
    }
}

  3. 实现图片文件上传控制层

    /**
 * 文件上传控制器
 */
@RestController
@RequestMapping("/upload")
@Slf4j
public class UploadController {
    @Autowired
    private IUploadService uploadService;
    @PostMapping("/{path}")
    public JsonResult<UrlVO> uploadFile(@PathVariable String path, MultipartFile file){
        log.info("开始处理[文件上传]的请求,参数{}",path);
        log.info("开始处理[文件上传]的请求,参数{}",file);
        UrlVO urlVO = uploadService.uploadFile(path,file);
        return JsonResult.ok(urlVO);
    }
}

  4. 设置静态资源目录显示远程图片

    在Spring Boot中，可以通过在application.yml文件中配置静态资源目录来设置静态资源的路径。设置完成后，Spring Boot就会将静态资源路径添加到默认的静态资源路径中，使得我们可以访问这些资源。

      # 文件静态资源映射
spring:
  web:
    resources:
      static-locations: classpath:/static/, file:${upload.dir}

    在上述配置中，classpath:/static/表示静态资源存放在src/main/resources/static目录下，而file:/D:/upload/dir表示静态资源还存放在D:/upload目录下

@Data
@AllArgsConstructor
@NoArgsConstructor

    private String url;
